# Adding Groups

Admin groups allow you to organize administrators by their roles and responsibilities, and to assign specific permissions to each group. This ensures that admins only have access to the features and data necessary for their job, supporting both security and operational efficiency.

---

## How to Create an Admin Group

1. **Navigate to the Groups Section**  
   Access the Admin Groups page and click the help link for this guide.

2. **Start a New Group**  
   Click the **New Group** button to begin creating a group.

   ![New Group Button](https://s3.amazonaws.com/helpscout.net/docs/assets/5b4c4e430428631d7a88ed10/images/5b4c4fbc0428631d7a88ed8b/img-28331-1531727780-1986183645.png)

3. **Enter Group Details**  
   - **Name**: Give your group a clear, descriptive name.
   - **Description**: Briefly describe the group’s purpose or scope.
   - **Permissions**: Use the checkboxes to enable or disable access to various sections of the system. For example, you may allow some groups to view reports but not edit billing settings.

   ![Group Permissions](https://s3.amazonaws.com/helpscout.net/docs/assets/5b4c4e430428631d7a88ed10/images/5b4c4fbc2c7d3a03f89c97ae/img-28331-1531727780-654690696.png)

   > **Tip:** When granting "Edit" rights, you should also enable "View" rights for the same section.

4. **Create the Group**  
   Once you have configured the group’s permissions, click **Create Group**. You will be taken to the Edit Group page for further adjustments if needed.

5. **Assign Admins to Groups**  
   You can assign admins to groups when creating or editing their user profiles. This controls their access throughout the system.

---

Admin groups are a powerful way to manage access and maintain security. Review group memberships regularly to ensure admins have the appropriate permissions.
